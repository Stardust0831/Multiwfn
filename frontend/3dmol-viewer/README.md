# Multiwfn 3Dmol Viewer Prototype

This is a static 3Dmol.js prototype for checking the visualization direction
before integrating more deeply with Multiwfn.

Implemented:

- Load molecular structures from XYZ/PDB/SDF/MOL2/PQR text files.
- Load Gaussian cube files generated by Multiwfn or other programs.
- Keep separate HOMO and LUMO cube slots so frontier orbitals can be compared
  in one scene.
- Show positive and negative orbital isosurfaces with adjustable isovalue and
  opacity.
- Extract atom positions from a cube file when no separate structure is loaded.

Recommended manual check:

1. Load the structure file exported from Multiwfn or the quantum chemistry
   program.
2. Choose the HOMO slot and load the HOMO cube.
3. Choose the LUMO slot and load the LUMO cube.
4. Adjust isovalue and opacity until the phase coloring and molecular overlay
   are easy to inspect.

The prototype does not invoke Multiwfn yet. It is intentionally a frontend-only
viewer for manually checking whether 3Dmol.js is adequate before adding a
bridge layer.

3Dmol.js references used while implementing this prototype:

- Official include snippet: `https://3Dmol.org/build/3Dmol-min.js`
- `GLViewer.addModel(data, format, options)` for molecular structures.
- `VolumeData(data, "cube")` and `GLViewer.addIsosurface(...)` for cube
  isosurfaces.

Run locally from this directory with any static server, for example:

```sh
python3 -m http.server 8080
```

Then open `http://127.0.0.1:8080/`.
